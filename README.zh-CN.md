# Mesh Annotation Layers 1.3 简体中文用户指南

[English README](README.md) · [安装与开发（英文）](INSTALL.md) ·
[快速参考（英文）](QUICK_REFERENCE.md) · [常见问题（英文）](FAQ.md)

Mesh Annotation Layers 是一个 Blender 网格标注扩展。它可以把面、边和点组织成
带颜色的标注图层，并随 `.blend` 文件保存。分配操作在编辑模式中完成，标注仍可显示
在物体、雕刻、权重绘制、顶点绘制和纹理绘制模式中。

## 适用范围

- 正式版本：1.3.0
- 清单最低版本：Blender 4.2
- 当前实机验证：Blender 5.1.2
- 无第三方 Python 依赖
- 不申请网络、文件系统、剪贴板、摄像头或麦克风权限

清单校验不等于运行时测试。若要部署到 Blender 4.2，建议在目标版本中完成一次安装、
保存重开和基本工作流验证。

## 安装与启用

1. 获取由 Blender 官方扩展命令构建的 `mesh_annotation_layers-1.3.0.zip`。
2. 打开 **编辑 > 偏好设置 > 获取扩展**。
3. 打开仓库菜单，选择 **从磁盘安装...**。
4. 选择 ZIP；如未自动启用，再启用 **Mesh Annotation Layers**。
5. 选择一个网格物体，在 3D 视图按 **N**，打开 **网格标注** 页签。

不要解压 ZIP，也不要直接安装 GitHub 的源码归档。可安装的扩展包必须在根目录包含
`blender_manifest.toml`、`__init__.py`、全部模块和 `LICENSE`。

## 快速开始

1. 选择一个网格物体并进入编辑模式。
2. 在侧栏的 **网格标注** 中选择面、边或点；对应页签也会切换 Blender 的网格选择模式。
3. 点击 **+** 新建图层，可以修改名称、颜色、顺序和可见性。
4. 选择网格元素，点击 **添加选中**，将它们加入活动图层。
5. 需要连续路径时使用 **添加循环**；选择必须连通且足以消除歧义。
6. 使用 **选中 → 新图层** 或 **循环 → 新图层**，一次完成创建与分配。

一个网格元素可以同时属于多个图层。显示时以顺序最高的可见图层为准；从活动图层
移除只会移除该图层，不会破坏同一元素上的其他标注。

## 图层和分配操作

| 控件或操作 | 作用 |
| --- | --- |
| `+` | 新建图层 |
| `−` | 删除活动图层及其分配 |
| `▲ / ▼` | 调整覆盖显示顺序 |
| 眼睛图标 | 显示或隐藏单个图层 |
| 颜色 | 修改图层颜色 |
| 添加选中 | 把当前选区加入活动图层 |
| 添加循环 | 推导面环、边环或点路径并加入活动图层 |
| 选中图层元素 | 选择活动图层中的全部元素 |
| 从选择拾取 | 激活当前选区中使用最多的图层 |
| 从活动图层移除 | 只移除活动图层，保留其他重叠分配 |
| 只移除顶层 | 只移除顺序最高的分配 |
| 移除全部 | 清除选中元素的所有标注 |

点模式还支持按连接边数量（Valence）批量标注。面模式可以把活动图层或全部面图层的
外边界标记为 UV 缝合边。

## 编辑模式右键菜单

在 3D 视图中右键，打开 **网格标注**。菜单会跟随当前的面、边或点选择模式，并直接
提供以下常用操作：

- 把选中元素或循环加入活动图层；
- 从选中元素或循环直接创建新图层；
- 选择另一个已有目标图层；
- 从活动图层、顶层或全部图层移除选中元素。

只有“选择另一个已有图层”会再打开一层子菜单。

## 显示设置

侧栏的 **显示** 子面板包括：

- 总透明度；
- 穿透网格显示；
- 面、边、点的表面偏移；
- 边线宽度和端点缩短；
- 点大小；
- 调试输出。

**覆盖显示** 可以关闭全部标注绘制，**仅活动图层** 可以只显示每种元素类型的活动
图层。图层颜色不会随 Blender 选择高亮而改变。

## 共享 Mesh 数据

图层设置和持久分配属于各个 Object，而用于跟随拓扑和撤销历史的 BMesh 栈属于 Mesh。
多个 Object 共享同一个 Mesh 时，标注写入会变成只读，避免一个物体覆盖另一个物体的
标注。

使用 **将网格转为单用户**。扩展会先验证 Object 的 JSON、当前拓扑和 Mesh 自定义数据：

- **仅在验证通过时保留**：只有证明完全一致才自动分离；
- **保留物体分配**：明确按当前元素索引保留 Object 的分配；
- **丢弃分配**：保留图层定义，但清除无法验证的元素分配。

未经验证的共享分配不会绘制、选择，也不会被自动恢复。Fake User 和 Extra User 不会被
误判成第二个真实 Object 用户。

## 数据安全与性能

- 经过校验的 Object JSON 是持久数据源；BMesh 自定义层负责让标注跟随拓扑和撤销历史。
- 重叠图层使用带版本、校验和的紧凑二进制栈；旧逗号格式只读取、不再写入。
- Blender 的 BMesh 字符串值上限为 255 字节。扩展会在修改数据前预检，容量不足时整次
  操作取消，不留下半写入状态。
- 新建 BMesh 栈只初始化已有分配；强制重建时才完整清除未分配元素。
- 普通状态指纹只处理已标注元素；共享 Mesh 的可信边界会额外只读核对完整栈。
- 显式选择、拾取和写入会立即协调 BMesh，避免同数量拓扑替换导致标签串位。
- JSON、求值几何和 GPU 批次使用有界缓存；无关场景更新和单纯样式变化不会重建全部几何。
- 视口绘制只读；拓扑协调由独立定时器完成，并在交互停止后安排最终刷新。

## 常见问题

| 现象 | 检查或处理 |
| --- | --- |
| 看不到标注 | 检查覆盖显示、图层眼睛、透明度、活动模式和图层分配 |
| 添加操作被取消 | 确认处于编辑模式、已有活动图层且选区非空 |
| 循环被拒绝 | 使用连通且没有歧义的面、边或点种子 |
| 控件被锁定 | Mesh 正被多个 Object 使用；先转为单用户 |
| 提示重叠容量不足 | 减少同一元素上的图层数量 |
| 拓扑修改后共享标注消失 | 分配已被隔离；使用单用户恢复对话框明确处理 |
| 修改源码后界面仍是旧版本 | 使用 **F3 > Reload Scripts** |
| ZIP 无法安装 | 确认使用 Blender 官方构建产物，而不是源码归档 |

Undo/Redo 会清除派生缓存并从 Blender 历史状态恢复所有权。若某个特殊 modifier 栈生成
了不寻常拓扑，请在目标 Blender 版本上验证；Subdivision Surface 和 Mirror 有专用映射
路径，其他情况使用通用回退映射。

## 语言设置

打开 **编辑 > 偏好设置 > 插件 > Mesh Annotation Layers**，可选择：

- **自动**：中文 Blender 界面使用中文，其他语言回退为英文；
- **英文**；
- **中文**。

## 开发、测试与构建

推荐把源码保存在普通 Git 工作目录，再把 `mesh_annotation_layers` 目录链接到 Blender
的本地扩展仓库。修改 Python 后使用 **F3 > Reload Scripts**。完整开发环境步骤见
[INSTALL.md](INSTALL.md)。

提交前至少运行：

```text
python -m compileall -q mesh_annotation_layers tests
python -m unittest discover -s tests -p "test_*.py" -v
blender --background --factory-startup --python tests/blender_smoke.py
blender --command extension validate mesh_annotation_layers
```

正式包只使用 Blender 官方扩展命令构建：

```text
blender --command extension build --source-dir mesh_annotation_layers --output-dir dist
blender --command extension validate dist/mesh_annotation_layers-1.3.0.zip
```

项目不维护第二套打包器。开发者还应阅读 [ARCHITECTURE.md](ARCHITECTURE.md)、
[CONTRIBUTING.md](CONTRIBUTING.md) 和发布测试清单
[.github/TESTING_CHECKLIST.md](.github/TESTING_CHECKLIST.md)。这些开发文档目前以英文为准。
